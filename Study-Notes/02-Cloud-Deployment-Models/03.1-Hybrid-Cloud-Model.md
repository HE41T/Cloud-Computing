# Hybrid Cloud Model (Data-Centric Perspective)

## Hybrid Cloud คืออะไร
Hybrid Cloud คือการผสมผสานระหว่าง
Public Cloud และ Private Cloud
โดยแยกการใช้งานตามระดับความสำคัญของข้อมูล

> ข้อมูลไม่เหมือนกัน Cloud ไม่ควรเหมือนกัน

---

## แนวคิดหลักของ Hybrid Cloud
- Sensitive Data อยู่ Private Cloud
- Non-Sensitive Data อยู่ Public Cloud
- เชื่อมต่อกันด้วย Secure Network

---

## ตัวอย่างการจัดวางข้อมูล

| ประเภทข้อมูล | Cloud |
|-------------|-------|
| Website Content | Public Cloud |
| User Session | Public Cloud |
| Transaction Data | Private Cloud |
| Personal Data | Private Cloud |
| Backup / DR | Public Cloud (Encrypted) |

---

## ทำไมองค์กรขนาดใหญ่เลือก Hybrid Cloud

### ด้านข้อมูล
- ลดความเสี่ยงของข้อมูลสำคัญ
- รองรับ Compliance

### ด้านธุรกิจ
- ลดต้นทุนรวม
- เพิ่มความยืดหยุ่น
- รองรับการเติบโตในอนาคต

---

## Use Case ที่พบบ่อย
- Digital Transformation
- Cloud Bursting
- Disaster Recovery
- Gradual Migration

---

## ความสำคัญเชิงกลยุทธ์
- Balance ระหว่าง Cost และ Security
- ลด Vendor Lock-in
- รองรับการตรวจสอบและ Audit

---

## ประโยคอธิบายลูกค้า
> Hybrid Cloud คือการวางข้อมูลให้ถูกที่  
> เพื่อให้ธุรกิจเดินได้เร็วและปลอดภัยพร้อมกัน
