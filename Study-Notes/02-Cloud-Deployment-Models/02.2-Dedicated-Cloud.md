# Dedicated Cloud (Enterprise & Data-Centric Perspective)

## Dedicated Cloud คืออะไร
Dedicated Cloud คือรูปแบบ Cloud ที่
**โครงสร้างพื้นฐานทางกายภาพ (Physical Infrastructure)**
ถูกจัดสรรให้ลูกค้ารายเดียวโดยเฉพาะ  
ไม่ใช้ Hardware ร่วมกับลูกค้ารายอื่น

แต่ยังคงคุณสมบัติแบบ Cloud เช่น
- On-demand
- Elastic
- API / Automation
- Pay-as-you-use (ตามสัญญา)

> สรุปสั้น ๆ  
> **ไม่แชร์เครื่อง แต่ยังได้ประสบการณ์แบบ Cloud**

---

## Dedicated Cloud แตกต่างจาก Public / Private Cloud อย่างไร

| ประเด็น | Public Cloud | Dedicated Cloud | Private Cloud |
|--------|-------------|----------------|---------------|
| ใช้ Hardware ร่วม | ✅ | ❌ | ❌ |
| การแยก | Logical Isolation | Physical Isolation | Physical Isolation |
| ใครเป็นเจ้าของ Hardware | Provider | Provider | องค์กร |
| การดูแล Infra | Provider | Provider | องค์กร |
| ความยืดหยุ่น | สูง | สูง | ปานกลาง |
| ค่าใช้จ่าย | ต่ำ | ปานกลาง–สูง | สูง |

> Dedicated Cloud = Private Cloud ที่เช่าจาก Provider

## มุมมองด้านข้อมูล (Data Perspective)

### ข้อมูลที่เหมาะกับ Dedicated Cloud
- Confidential Data
- Sensitive Personal Data
- Financial / Transaction Data
- Health Data
- Enterprise Core System

### ตัวอย่างข้อมูล
- Core Banking Transaction
- Payment Information
- ERP / SAP Data
- Medical Record (Encrypted)
- Customer PII (PDPA / GDPR)

---

## ทำไมข้อมูลเหล่านี้ต้องใช้ Dedicated Cloud
1. **Physical Isolation** – ไม่แชร์ Hardware
2. **ลดความเสี่ยงจาก Misconfiguration**
3. **ง่ายต่อ Compliance และ Audit**
4. **ควบคุม Data Residency ได้ชัดเจน**
5. **Performance คงที่ (Predictable Performance)**

---

## Dedicated Cloud สำคัญต่อธุรกิจองค์กรอย่างไร

### ด้านความปลอดภัย
- ลดความเสี่ยง Data Breach
- รองรับ Regulatory Requirement
- เพิ่ม Trust ให้ลูกค้าและคู่ค้า

### ด้านธุรกิจ
- ไม่ต้องลงทุน Hardware เอง (CAPEX → OPEX)
- ได้ SLA ระดับ Enterprise
- รองรับระบบ Mission-Critical
- ขยายระบบได้เร็วกว่า On-Prem

## Dedicated Cloud vs Logical Isolation

| ประเด็น | Logical Isolation | Dedicated Cloud |
|--------|------------------|----------------|
| แยกด้วย Software | ✅ | ❌ |
| แยกด้วย Hardware | ❌ | ✅ |
| ความเสี่ยงจาก Config | สูงกว่า | ต่ำกว่า |
| Cost | ต่ำ | สูง |
| เหมาะกับ Data Sensitivity | ปานกลาง | สูง |

---

## ข้อจำกัดของ Dedicated Cloud
- ค่าใช้จ่ายสูงกว่า Public Cloud
- Scale ช้ากว่า Shared Environment
- ต้องวาง Capacity Plan
- Vendor Lock-in บางส่วน

---

## Best Practice การใช้งาน Dedicated Cloud
1. ใช้กับ Workload ที่จำเป็นจริง
2. ผสานกับ Hybrid Cloud
3. แยก Management Network
4. ใช้ Encryption แม้จะเป็น Dedicated
5. มี Monitoring และ Audit Log
