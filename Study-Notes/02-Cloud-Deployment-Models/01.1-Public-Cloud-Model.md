# Public Cloud Model (Data-Centric Perspective)

## Public Cloud คืออะไร
Public Cloud คือรูปแบบ Cloud ที่โครงสร้างพื้นฐาน (Infrastructure)
เป็นของผู้ให้บริการ Cloud และเปิดให้ลูกค้าหลายองค์กรใช้งานร่วมกัน
โดยแยกข้อมูลออกจากกันด้วย Logical Isolation

> Public Cloud ไม่ได้หมายถึงข้อมูลเปิดเผยสาธารณะ  
> แต่หมายถึงโครงสร้างที่ใช้ร่วมกัน

---

## ประเภทข้อมูลที่เหมาะกับ Public Cloud

### ข้อมูลที่เหมาะสม
- Public Data (ข้อมูลเว็บไซต์, ข่าว, ราคา)
- Internal Data (ข้อมูลภายในองค์กร)
- Confidential Data (ถ้ามีการควบคุมและเข้ารหัส)

### ตัวอย่าง
- Website Content
- Product Catalog
- Log / Monitoring
- Analytics Data
- User Profile (Encrypted)

---

## ข้อมูลที่ไม่เหมาะกับ Public Cloud
- National Security Data
- Core Financial Ledger (บางอุตสาหกรรม)
- Health Record แบบ Raw ที่ไม่เข้ารหัส
- ข้อมูลที่กฎหมายบังคับให้เก็บ On-Prem

---

## การป้องกันข้อมูลใน Public Cloud
Cloud Provider มีเครื่องมือด้านความปลอดภัย เช่น
- Encryption at Rest
- Encryption in Transit
- IAM / RBAC
- Audit Log
- Network Isolation

ความปลอดภัยของข้อมูลขึ้นอยู่กับ **การออกแบบระบบขององค์กร**

---

## ความสำคัญเชิงธุรกิจ
- ลดต้นทุนด้าน Infrastructure
- Scale ได้ตามปริมาณข้อมูล
- รองรับ Big Data / AI / Analytics
- เปิดระบบใหม่ได้รวดเร็ว

---

## ตัวอย่างผู้ให้บริการ
- AWS
- Google Cloud
- Microsoft Azure
- Alibaba Cloud

---

## ประโยคอธิบายลูกค้า
> Public Cloud เหมาะกับข้อมูลที่ต้องการความยืดหยุ่นสูง  
> และสามารถควบคุมความปลอดภัยได้ด้วยการออกแบบระบบ
