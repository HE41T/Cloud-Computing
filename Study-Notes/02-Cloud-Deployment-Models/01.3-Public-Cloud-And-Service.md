# Public Cloud Model เมื่อใช้งานร่วมกับ Service Model

อธิบายการให้บริการ การทำงาน และสิ่งที่ลูกค้าจะได้รับจาก Public Cloud เมื่อใช้งานร่วมกับ IaaS, PaaS และ SaaS

---

## 1. แนวคิดของ Public Cloud (ในมุมการใช้งานจริง)

**Public Cloud** คือรูปแบบ Cloud ที่ผู้ให้บริการเป็นเจ้าของโครงสร้างพื้นฐานทั้งหมด และเปิดให้ลูกค้าหลายรายใช้งานร่วมกันผ่าน Internet โดยมีลักษณะสำคัญดังนี้

* ใช้ทรัพยากรร่วมกัน (Multi-Tenant)
* ไม่ต้องลงทุน Hardware
* ใช้งานได้ทันที
* คิดค่าบริการตามการใช้งานจริง (Pay-as-you-use)

> Public Cloud ไม่ได้ขายเครื่อง Server แต่ขายเป็น “ระดับของบริการ” ผ่าน Service Model

---

## 2. Public Cloud + IaaS (Infrastructure as a Service)

### รูปแบบการให้บริการ

IaaS คือการให้บริการโครงสร้างพื้นฐานระดับล่าง โดยผู้ให้บริการ Cloud จะจัดเตรียม

* Data Center
* Physical Server
* Storage
* Network
* Virtualization / Hypervisor

ลูกค้าเช่าใช้งานในรูปแบบ **Virtual Machine (VM)**

---

### การทำงาน (เชิงลึก)

1. ลูกค้าเลือกสเปก VM (CPU, RAM, Disk, Network)
2. Cloud Provider สร้าง VM บน Physical Server
3. ลูกค้าเลือกและติดตั้ง OS (Linux / Windows)
4. ลูกค้าติดตั้ง Software และ Application เอง
5. ลูกค้าดูแลระบบทั้งหมดเหมือน Server ของตนเอง

> การใช้งาน IaaS ใกล้เคียงกับ On-Premise มาก แต่ไม่ต้องดูแล Hardware

---

### สิ่งที่ลูกค้าได้รับ

* Server พร้อมใช้งานภายในไม่กี่นาที
* ปรับสเปกได้ตลอดเวลา (Scale Up / Down)
* ควบคุม OS และระบบได้เต็มที่
* รองรับ High Availability และ Disaster Recovery

---

### ความรับผิดชอบ (Shared Responsibility)

| ส่วน           | ผู้ดูแล        |
| -------------- | -------------- |
| Data Center    | Cloud Provider |
| Hardware       | Cloud Provider |
| Virtualization | Cloud Provider |
| OS             | ลูกค้า         |
| Patch OS       | ลูกค้า         |
| Application    | ลูกค้า         |
| Data           | ลูกค้า         |

---

### เหมาะกับการใช้งาน

* ระบบ Legacy
* ระบบที่ต้องควบคุม OS เอง
* องค์กรที่มี System / Infra Team

---

## 3. Public Cloud + PaaS (Platform as a Service)

### รูปแบบการให้บริการ

PaaS เป็นการยกระดับจาก IaaS โดยผู้ให้บริการ Cloud จะดูแลตั้งแต่

* Infrastructure
* OS
* Runtime / Middleware

ลูกค้าโฟกัสเฉพาะ **Application**

---

### การทำงาน (อธิบายกรณี “ลูกค้าไม่ต้องดูแล OS”)

เมื่อใช้งาน PaaS:

* ลูกค้าไม่ต้องสร้าง VM
* ลูกค้าไม่ต้องติดตั้ง OS
* ลูกค้าไม่ต้อง Patch OS
* ลูกค้าไม่ต้องดูแล Runtime (เช่น Java, Node.js)

**Cloud Provider จะเป็นผู้ดูแลทั้งหมด** โดย

* ใช้ OS Image มาตรฐาน
* Patch Security อัตโนมัติ
* Monitor ช่องโหว่
* Replace Instance เมื่อเกิดปัญหา

ลูกค้ามีหน้าที่เพียง

* เขียน Code
* Deploy Application
* จัดการ Data และ Logic ของระบบ

---

### Flow การทำงาน

1. ลูกค้าเขียน Application
2. Push Code ขึ้น Platform
3. Platform สร้าง Environment ให้อัตโนมัติ
4. ระบบจัดการ Load Balancer และ Auto Scaling
5. Application พร้อมใช้งานทันที

---

### สิ่งที่ลูกค้าได้รับ

* ลดภาระงาน System Administration
* Deploy Application ได้รวดเร็ว
* มาตรฐานความปลอดภัยสูง
* รองรับการ Scale อัตโนมัติ

---

### ความรับผิดชอบ

| ส่วน             | ผู้ดูแล        |
| ---------------- | -------------- |
| Infrastructure   | Cloud Provider |
| OS               | Cloud Provider |
| Runtime          | Cloud Provider |
| Scaling          | Cloud Provider |
| Application Code | ลูกค้า         |
| Data             | ลูกค้า         |

---

### เหมาะกับการใช้งาน

* Developer / DevOps Team
* Web Application / API
* Microservices
* Startup

---

## 4. Public Cloud + SaaS (Software as a Service)

### รูปแบบการให้บริการ

SaaS คือการให้บริการ Software สำเร็จรูป ผู้ให้บริการดูแลทุกอย่างตั้งแต่ต้นจนจบ

ลูกค้าไม่ต้องรู้ว่า

* ใช้ OS อะไร
* รันอยู่บน Server ไหน
* ใช้ Infrastructure แบบใด

---

### การทำงาน

1. ผู้ให้บริการพัฒนาและดูแล Software
2. Deploy Software บน Cloud Infrastructure
3. ดูแล OS, Application, Security และ Update
4. ลูกค้าใช้งานผ่าน Browser หรือ Application

> ลูกค้าเป็นเพียงผู้ใช้งาน ไม่ต้องดูแลระบบใด ๆ

---

### สิ่งที่ลูกค้าได้รับ

* ใช้งานได้ทันที
* Update ระบบอัตโนมัติ
* ไม่ต้องมี IT Team
* ค่าใช้จ่ายคาดการณ์ได้ง่าย

---

### ความรับผิดชอบ

| ส่วน           | ผู้ดูแล  |
| -------------- | -------- |
| Infrastructure | Provider |
| OS             | Provider |
| Application    | Provider |
| Security       | Provider |
| การใช้งาน      | ลูกค้า   |

---

### เหมาะกับการใช้งาน

* ผู้ใช้งานทั่วไป
* องค์กรที่ไม่ต้องการดูแลระบบเอง
* ธุรกิจที่ต้องการความรวดเร็ว

---

## 5. เปรียบเทียบภาพรวม (เน้นการดูแล OS)

| Service Model | ลูกค้าดูแล OS | สิ่งที่ลูกค้าทำ    |
| ------------- | ------------- | ------------------ |
| IaaS          | ต้องดูแลเอง   | OS และ Application |
| PaaS          | ไม่ต้องดูแล   | Application        |
| SaaS          | ไม่เกี่ยวข้อง | ใช้งาน Software    |

---

## 6. สรุปแนวคิดแบบเข้าใจง่าย

* **IaaS**: ได้ Server แต่ต้องดูแลระบบเอง
* **PaaS**: ไม่ต้องดูแล OS โฟกัสที่ Application
* **SaaS**: ใช้งานอย่างเดียว ไม่ต้องดูแลอะไร

---
