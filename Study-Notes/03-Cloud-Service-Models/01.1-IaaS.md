# Infrastructure as a Service (IaaS)

## IaaS คืออะไร
Infrastructure-as-a-Service (IaaS) คือรูปแบบบริการคลาวด์ที่ให้ผู้ใช้งานสามารถเข้าถึงทรัพยากรพื้นฐานด้านโครงสร้างพื้นฐาน เช่น

- เครื่องเซิร์ฟเวอร์จริง (Physical Machines)
- เครื่องเสมือน (Virtual Machines)
- ระบบจัดเก็บข้อมูลแบบเสมือน (Virtual Storage)

นอกจากทรัพยากรพื้นฐานแล้ว IaaS ยังให้บริการเพิ่มเติมดังนี้

- ดิสก์จัดเก็บข้อมูลของเครื่องเสมือน (Virtual Machine Disk Storage)
- เครือข่ายเสมือน (VLANs)
- Load Balancer
- IP Address
- Software Bundles

ทรัพยากรทั้งหมดนี้ถูกให้บริการแก่ผู้ใช้งานผ่านเทคโนโลยี **Server Virtualization**  
ผู้ใช้งานสามารถเข้าถึงและใช้งานทรัพยากรเหล่านี้ได้เสมือนเป็นเจ้าของเอง

---

## ประโยชน์ของ IaaS (Benefits)

IaaS ช่วยให้ผู้ให้บริการคลาวด์สามารถกระจายโครงสร้างพื้นฐานผ่านอินเทอร์เน็ตได้อย่างคุ้มค่า  
ประโยชน์หลักของ IaaS มีดังนี้

- ควบคุมทรัพยากรการประมวลผลได้อย่างเต็มที่ผ่านสิทธิ์ผู้ดูแลระบบ (Admin)
- เช่าใช้งานฮาร์ดแวร์คอมพิวเตอร์ได้อย่างยืดหยุ่นและมีประสิทธิภาพ
- รองรับการย้ายระบบ (Portability) และทำงานร่วมกับระบบเดิม (Legacy Applications)

---

## การควบคุมทรัพยากรด้วยสิทธิ์ผู้ดูแลระบบ (Full Control)

IaaS อนุญาตให้ผู้ใช้งานเข้าถึงทรัพยากรการประมวลผลผ่านสิทธิ์ผู้ดูแลระบบของเครื่องเสมือนได้ เช่น

- ผู้ใช้งานสามารถส่งคำสั่งไปยังผู้ให้บริการคลาวด์เพื่อ:
  - สร้างหรือรัน Virtual Machine
  - จัดเก็บข้อมูลบน Cloud Server

- ผู้ใช้งานสามารถส่งคำสั่งไปยัง Virtual Machine ของตนเองเพื่อ:
  - เปิดใช้งาน Web Server
  - ติดตั้งแอปพลิเคชันใหม่

---

## การเช่าฮาร์ดแวร์ที่ยืดหยุ่นและมีประสิทธิภาพ

ทรัพยากรของ IaaS เช่น

- Virtual Machines
- Storage
- Bandwidth
- IP Address
- Monitoring Services
- Firewall

ถูกให้บริการในรูปแบบ **เช่าใช้งาน (Rent)**  
ค่าใช้จ่ายจะขึ้นอยู่กับระยะเวลาที่ผู้ใช้งานใช้ทรัพยากรนั้น

ด้วยสิทธิ์ผู้ดูแลระบบ ผู้ใช้งานสามารถ:
- ติดตั้งซอฟต์แวร์ใดก็ได้
- ใช้ระบบปฏิบัติการแบบกำหนดเอง (Custom OS)

---

## การรองรับระบบเดิม (Portability & Interoperability)

IaaS รองรับการทำงานร่วมกับแอปพลิเคชันและระบบเดิม (Legacy Systems)

ตัวอย่าง:
- Web Server
- Mail Server

ซึ่งเดิมเคยรันบนเซิร์ฟเวอร์ขององค์กร สามารถย้ายมารันบน Virtual Machine ใน IaaS Cloud ได้โดยไม่ต้องแก้ไขระบบมาก

---

## ปัญหาและข้อจำกัดของ IaaS (Issues)

IaaS มีปัญหาร่วมกับ PaaS และ SaaS เช่น
- การพึ่งพาเครือข่าย (Network Dependence)
- ความเสี่ยงจากการใช้งานผ่านเว็บเบราว์เซอร์

และยังมีปัญหาเฉพาะของ IaaS ดังนี้

### 1. ความเข้ากันได้กับช่องโหว่ของระบบเดิม
เนื่องจาก IaaS อนุญาตให้ผู้ใช้งานรันซอฟต์แวร์รุ่นเก่า  
จึงทำให้มีความเสี่ยงด้านความปลอดภัยจากช่องโหว่ของซอฟต์แวร์เหล่านั้น

### 2. Virtual Machine Sprawl
ผู้ใช้งานสามารถ:
- เปิด (Running)
- พัก (Suspended)
- ปิด (Off)

เครื่องเสมือนได้อย่างอิสระ  
ทำให้บาง VM อาจไม่ได้รับการอัปเดตด้านความปลอดภัยอย่างสม่ำเสมอ  
แม้ผู้ให้บริการจะสามารถอัปเดตอัตโนมัติได้ แต่กระบวนการมีความซับซ้อน

### 3. ความแข็งแรงของการแยก VM
IaaS ใช้ **Hypervisor** เพื่อแยกสภาพแวดล้อมของลูกค้าแต่ละราย  
Hypervisor คือซอฟต์แวร์ที่ช่วยแบ่งเครื่องจริงออกเป็นหลายเครื่องเสมือน  
ความปลอดภัยขึ้นอยู่กับความแข็งแรงของการแยกในระดับนี้

### 4. การลบข้อมูล (Data Erase Practices)
เมื่อผู้ใช้งานคืนทรัพยากร:
- ผู้ให้บริการต้องมั่นใจว่า
- ลูกค้ารายถัดไปจะไม่สามารถเข้าถึงข้อมูลคงค้างของลูกค้ารายก่อนหน้าได้

---

## คุณลักษณะของ IaaS (Characteristics)

ลักษณะสำคัญของ IaaS มีดังนี้

- มี Virtual Machine ที่ติดตั้งซอฟต์แวร์มาให้ล่วงหน้า
- รองรับระบบปฏิบัติการ เช่น
  - Windows
  - Linux
  - Solaris
- ทรัพยากรพร้อมใช้งานแบบ On-demand
- สามารถเก็บสำเนาข้อมูลไว้หลายตำแหน่ง
- ปรับเพิ่มหรือลดทรัพยากร (Scale Up / Scale Down) ได้อย่างง่ายดาย
